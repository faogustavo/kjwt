# RFC 7519 Compliance Checklist

This document tracks KJWT's compliance with [RFC 7519 (JWT)](https://datatracker.ietf.org/doc/html/rfc7519).

## Section 4 - JWT Claims

| Requirement | Status | Notes |
|---|---|---|
| `iss` (Issuer) claim | Supported | `builder.issuer()` / `parser.requireIssuer()` |
| `sub` (Subject) claim | Supported | `builder.subject()` / `parser.requireSubject()` |
| `aud` (Audience) claim | Supported | Single string or JSON array; normalized to `Set<String>` on parse |
| `exp` (Expiration Time) claim | Supported | Auto-validated on parse; `builder.expiration()` / `expiresIn()` |
| `nbf` (Not Before) claim | Supported | Auto-validated on parse; `builder.notBefore()` / `notBeforeNow()` |
| `iat` (Issued At) claim | Supported | Stored and accessible; not validated (informational per RFC) |
| `jti` (JWT ID) claim | Supported | `builder.id()` / `randomId()`; no built-in uniqueness/replay check |
| Private/custom claims | Supported | `builder.claim(name, value)` with reified generics or explicit serializer |
| Unknown claims ignored | Supported | `ignoreUnknownKeys=true` in the internal JSON decoder |
| Duplicate claim names | Per-library | Handled by kotlinx.serialization (last value wins) |
| NumericDate (epoch seconds) | Supported | `InstantEpochSecondsSerializer` converts `kotlin.time.Instant` ↔ Unix epoch seconds |

## Section 5 - JOSE Header

| Requirement | Status | Notes |
|---|---|---|
| `alg` header parameter | Supported | Set automatically from the chosen `JwsAlgorithm` / `JweKeyAlgorithm` |
| `enc` header parameter (JWE) | Supported | Set automatically from the chosen `JweContentAlgorithm` |
| `typ` header parameter | Supported | Defaults to `"JWT"`; configurable via `builder.header { type = ... }` |
| `cty` header parameter | Supported | Configurable via `builder.header { contentType = ... }`; not validated on parse |
| `kid` header parameter | Supported | `builder.keyId(kid)` or `builder.header { keyId = ... }` |
| Unknown header parameters | Supported | Stored in `header.extra: Map<String, JsonElement>` |
| Nested JWT (`cty: "JWT"`) | Not validated | `cty` is stored but nested JWT unwrapping is not performed automatically |
| `crit` (Critical) header | Not implemented | Critical header parameters are not validated |

## Section 6 - Unsecured JWTs

| Requirement | Status | Notes |
|---|---|---|
| `alg=none` produces unsigned token | Supported | `builder.signWith(JwsAlgorithm.None)` |
| `alg=none` rejected by default | Supported | Parser throws `UnsupportedJwtException` unless `allowUnsecured(true)` |
| Opt-in to accept unsecured | Supported | `parser.allowUnsecured(true)` or `parser.noVerify()` |

## Section 7 - Creating and Validating JWTs

| Requirement | Status | Notes |
|---|---|---|
| Compact serialization (JWS) | Supported | `BASE64URL(header).BASE64URL(payload).BASE64URL(signature)` |
| Compact serialization (JWE) | Supported | `BASE64URL(header).BASE64URL(encKey).BASE64URL(iv).BASE64URL(ct).BASE64URL(tag)` |
| Base64url encoding (no padding) | Supported | `Base64Url.kt` uses `kotlin.io.encoding.Base64` without padding |
| UTF-8 JSON payload | Supported | `kotlinx-serialization` with JSON format |
| Signature verification | Supported | Via `cryptography-kotlin`; throws `SignatureException` on failure |
| Authentication tag verification (JWE) | Supported | AES-GCM tag and CBC-HMAC tag both verified; throws `SignatureException` on failure |
| `exp` validation | Supported | Throws `ExpiredJwtException` |
| `nbf` validation | Supported | Throws `PrematureJwtException` |
| Clock skew tolerance | Supported | `parser.clockSkew(seconds)` (default: 0 - strict) |
| Audience validation | Supported | `parser.requireAudience(value)` - checks set membership |
| Custom claim validation | Supported | `parser.require(name, value)` |

## Section 8 - Algorithm Requirements (RFC 7518)

| Algorithm | Type | RFC Status | KJWT Status |
|---|---|---|---|
| `HS256` | JWS | Required | Supported |
| `none` | JWS | Required | Supported (opt-in) |
| `RS256` | JWS | Recommended | Supported |
| `ES256` | JWS | Recommended+ | Supported |
| `HS384`, `HS512` | JWS | Optional | Supported |
| `RS384`, `RS512` | JWS | Optional | Supported |
| `PS256`, `PS384`, `PS512` | JWS | Optional | Supported |
| `ES384`, `ES512` | JWS | Optional | Supported |
| `ECDH-ES` | JWE key | Recommended | Not supported - not available in cryptography-kotlin 0.5.0 |
| `RSA-OAEP` | JWE key | Optional | Supported |
| `RSA-OAEP-256` | JWE key | Optional | Supported |
| `dir` | JWE key | Optional | Supported |
| `A128KW`, `A192KW`, `A256KW` | JWE key | Optional | Not supported - AES Key Wrap not available in cryptography-kotlin 0.5.0 |
| `A128GCM`, `A192GCM`, `A256GCM` | JWE content | Recommended | Supported |
| `A128CBC-HS256` | JWE content | Required | Supported |
| `A192CBC-HS384`, `A256CBC-HS512` | JWE content | Optional | Supported |
